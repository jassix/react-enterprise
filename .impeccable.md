# Impeccable — Design Context

This file establishes the design direction for any work done in this repo via the Impeccable skill family (`/impeccable`, `/critique`, `/polish`, `/layout`, `/typeset`, `/colorize`, `/animate`, etc.). It is loaded before any design task to keep work grounded in the same point of view.

This monorepo is a **template that is lightly re-skinned per project** (primarily palette swaps, occasional font swaps). The design system below is the *baseline aesthetic* that survives across re-skins. Per-project re-skins should adjust palette and typography but preserve the geometry, rhythm, and motion language captured here.

---

## Design Context

### Users

Primarily product-builders and the end users of the products built on top of this template — knowledge workers and operators using **utility-grade web apps on desktop**, with the occasional editorial marketing surface. The center of gravity is the in-product UI: dashboards, tables, forms, settings, and dense data views consumed during sustained focused work, often for hours at a time. Marketing and landing pages are a secondary, editorial mode that links into that core.

### Brand Personality

**Three words: calm, geometric, deliberate.**

- **Calm** — nothing shouts. Surfaces and motion stay in the background until the user looks at them. The product is a tool that recedes, not a stage that performs.
- **Geometric** — rounded, considered, mathematical. Corners, ratios, and rhythm feel measured rather than improvised. Roundness is the personality of the geometry, not a decoration.
- **Deliberate** — every element earns its place. Restraint over abundance. When something is emphasized, it's because it actually matters; defaults are quiet so the loud moments can land.

Emotional goal: the user feels **competent and unhurried**. The interface should make focused work feel pleasant, not energetic. No urgency theatre, no novelty for its own sake.

### Aesthetic Direction

**Spectrum positioning**

- Minimal ↔ Maximal: **~20/100** — clean with breathing room, never austere. Generous space, soft elevation, room for the eye to rest.
- Editorial ↔ Utilitarian: **utilitarian core, editorial marketing.** App surfaces optimize for density and scannability; marketing surfaces shift to larger type, longer measures, and more compositional freedom. The Linear model.

**Visual touchstones (yes, do this)**

- **macOS Tahoe — minus the glass.** The geometry, the soft elevation, the rounded squircle-feel. Skip the literal liquid-glass material.
- **Apple** — generous whitespace, precise typography, restrained color, never crowded.
- **Linear** — fast, opinionated, dense without being noisy. Strong keyboard-first interaction model. Zero ornamental chrome.
- **Resend** — modern utility with quiet editorial polish on the marketing surface.
- **Mercury** — calm financial clarity, soft warmth in the neutrals, trust through restraint.

**Anti-references (don't do this)**

- **Glassmorphism / liquid glass.** Explicitly excluded — no decorative `backdrop-filter: blur` over translucent surfaces. Soft elevation through subtle shadows and tinted neutrals only.
- **Dashboard-template aesthetic.** No identical card grids, no stat-tile heroes (huge number + sparkline + delta chip), no border-left accent stripes, no rainbow chart palettes.
- **Generic shadcn/Tailwind starter look.** No grayscale neutrals on white with a single brand-blue accent and rounded-2xl cards everywhere.
- **AI-defaults palette.** No purple→blue gradients, no neon-on-near-black, no glowing borders.
- **Material Design.** No FABs, no bottom-sheet patterns transposed to web, no ink ripples.

**Theme**

- **Dark is primary** — the canonical surface, the one to screenshot for marketing, the one to design first.
- **Light is fully supported** — not an afterthought. Both pass WCAG 2.2 AA. Theme is a personal preference, not a context switch.
- Both themes use **OKLCH** with neutrals **tinted toward the brand hue** (chroma 0.005–0.01). Never pure black or pure white.

**Color**

- Per-project palette swap is expected and supported by the existing 23-color OKLCH token system in `packages/ui/foundation`.
- Default posture: **dominant neutral surfaces, one accent used sparingly.** 60-30-10 by visual weight.
- Status colors (positive, critical, caution, info) are part of the system but should never be louder than they need to be.

**Typography**

- Current `Figtree` is a placeholder from the template scaffold. When a project commits to its own typography, follow the font-selection procedure in the Impeccable SKILL — pair a distinctive display face with a refined body face, and avoid the reflex-fonts list (Inter, DM Sans, Plus Jakarta, Instrument family, Fraunces, etc.).
- Type scale: **fixed `rem` modular scale** for the app shell (Linear-style), **fluid `clamp()` headings** on marketing surfaces. Minimum 1.25 ratio between steps.
- Reserve all-caps for short labels only.

**Geometry**

- Rounded corners are the brand. Default radii lean generous (`md` and `lg`). Sharp corners are reserved for table cells, dividers, and surfaces that abut other rounded surfaces.
- Soft elevation: low-blur, low-opacity shadows that hint at depth without performing it. Stack at most two elevation levels in the same view.
- Spacing: 4pt base scale, semantic token names. Vary spacing for hierarchy — never the same padding everywhere.

**Motion**

- Motion is functional, not ornamental. State changes, entrances, and exits — yes. Idle decoration, ambient motion, and parallax — no.
- Easing: exponential ease-out for arrivals, linear or fast ease-in for departures. **No bounce, no elastic.**
- Durations live in `foundation/animations` — keep work inside the existing `fast / normal / slow` scale.
- **`prefers-reduced-motion: reduce` is honored everywhere.** Replace transforms with instant state changes; never just shorten the duration.

### Accessibility

- **WCAG 2.2 AA across the board** — non-negotiable. Both themes meet contrast targets for body text, UI text, and non-text components.
- **Keyboard-only support is non-negotiable.** Every interactive element is reachable, operable, and has a visible focus indicator. Focus order matches visual order. No keyboard traps.
- **`prefers-reduced-motion`** is respected at every level — recipes, page transitions, and any custom motion.
- Use Ark UI primitives (already in the stack) for any composite widget — they handle focus management, ARIA, and keyboard interaction correctly by default. Don't reinvent them.
- Color is never the sole carrier of meaning. Status, selection, and validation always pair color with shape, icon, or text.

### Design Principles

These five principles arbitrate every design decision. When two principles conflict, the earlier one wins.

1. **Recede, don't perform.** The interface is a tool, not a showpiece. Defaults are quiet; emphasis is rare and earned. If a decoration doesn't help the user complete the job, it doesn't ship.
2. **Soft geometry, exact rhythm.** Roundness is the personality; precision is the discipline. Rounded corners and soft elevation set the mood — but spacing, alignment, and type scale are mathematically consistent.
3. **Utility first, editorial when invited.** Product surfaces optimize for sustained focused work: dense, scannable, keyboard-fluent. Marketing surfaces are allowed to breathe, scale up, and use compositional freedom — but only on marketing surfaces.
4. **Dark is the canonical surface; light is a first-class peer.** Both themes are designed, not derived. Never ship a feature that only looks right in one of them.
5. **Accessibility is a constraint, not a feature.** WCAG 2.2 AA, full keyboard support, and reduced-motion compliance are baseline requirements that shape designs from the start — not audits performed at the end.
